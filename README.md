# Projektauftrag – SAFA Safran Website

## 1. Projekttitel

**SAFA-SAFRAN – Unternehmens- und Produktwebsite**

## 2. Auftraggeber

SAFA-SAFRAN

## 3. Teammitglieder

Emily, Sebi, Fawad

## 4. Projektbeschreibung / Ausgangslage

Derzeit verfügt SAFA-SAFRAN über keine zentrale, professionelle Website, auf der Kunden einen vollständigen Überblick über die angebotenen Produkte erhalten. Informationen über Safran, Produktqualität sowie Bundles (z. B. Safran in Kombination mit Kurkuma) sind nur eingeschränkt oder gar nicht online verfügbar. Dadurch gehen potenzielle Kunden und Verkaufsmöglichkeiten verloren.

## 5. Zielsetzung

Ziel dieses Projekts ist die Entwicklung einer übersichtlichen, modernen und benutzerfreundlichen Website für SAFA-SAFRAN. Die Website soll:

- Kunden einen klaren Überblick über alle Safran-Produkte und Bundles bieten
- den Online-Verkauf von Safran und Kurkuma-Bundles ermöglichen
- die hohe Qualität von 100 % reinem Safran von SAFA-SAFRAN professionell präsentieren
- Vertrauen und Markenwert aufbauen

## 6. Projektziele

- Erstellung einer funktionsfähigen Website bis Projektende
- Übersichtliche Darstellung aller Produkte inklusive Beschreibungen und Preisen
- Einfache Bestellmöglichkeit über die Website
- Klare, moderne Benutzeroberfläche (UI)
- Kompatibilität mit gängigen Browsern (Chrome, Edge, Firefox)

## 7. Nicht-Ziele

- Keine mobile App
- Kein externes Bezahlsystem (z. B. PayPal, Kreditkarte)
- Kein komplexes Warenwirtschaftssystem

## 8. Zielgruppe

- Privatkunden, die hochwertigen Safran kaufen möchten
- Gastronomiebetriebe
- Feinkost-Interessierte

## 9. Projektumfang / Funktionen

### Geplante Unterseiten

1. **Startseite** – Vorstellung von SAFA-SAFRAN und Produktqualität
2. **Produktseite** – Übersicht aller Safran-Produkte und Bundles inklusive Kurkuma
3. **Über uns** – Herkunft, Qualität und Philosophie von SAFA-SAFRAN
4. **Kontaktseite** – Kontaktformular und Standorte (z. B. Linz, Mailand, Afghanistan)
5. **Kundenbereich** – Registrierung, Login und Bestellübersicht
6. **Adminbereich** – Verwaltung von Produkten, Preisen und Bestellungen

### Funktionen

- Kundenregistrierung und Kunden-Login
- Admin-Login mit erweiterten Rechten
- Rollenverwaltung (Admin / Kunde)
- Produktverwaltung (Anlegen, Bearbeiten, Löschen)
- Warenkorb-Funktion
- Bestellübersicht für Kunden
- Kontaktformular
- Responsives Design

## 10. Technische Umsetzung

### Programmiersprachen & Technologien

- HTML
- CSS
- JavaScript
- Angular (Frontend)
- C# / ASP.NET (Backend)
- SQL-Datenbank

### Datenbank (SQL)

Die SQL-Datenbank speichert:

- Benutzerkonten (Kunden & Admins)
- Produktdaten
- Bestellungen
- Rollen und Zugriffsrechte

### Entwicklungsumgebung

- Visual Studio Code
- Visual Studio
- GitHub zur Versionsverwaltung

## 11. Systemarchitektur

Die Website wird als Webanwendung im Browser ausgeführt.
Die Benutzeroberfläche wird mit HTML, CSS und Angular umgesetzt.
JavaScript sorgt für Interaktionen.
Die Backend-Funktionen werden mit C# realisiert und greifen auf eine SQL-Datenbank zu.

## 12. Meilensteine

- M2 – Websitestart

- M4 – Produkte & Bundles

- M6 – Admin & Feinschliff

## 13. Risiken

- Zeitverzögerungen durch Designänderungen
- Unklare oder unvollständige Produktdaten

## 14. Erfolgskriterien

- Die Website ist übersichtlich und benutzerfreundlich
- Produkte sind klar und verständlich dargestellt
- Kunden können Safran-Produkte und Bundles problemlos bestellen
- Positive Rückmeldungen von Kunden
